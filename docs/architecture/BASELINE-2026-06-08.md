## 4. Can we start using the harness? — YES

**You've already opted in.** `CLAUDE.md` references the harness skills + threat-modeling triggers, and `ARCHITECTURE-INVARIANTS.md` exists at the repo root. The harness is a **forward gate** — it sequences brainstorm → plan (+threat-model/invariants) → execute (+tests) → shake-out → finish on your *next* feature. It does not require the existing 66k LOC to have been built with it.

**To actually use it on the next piece of work:**
1. Start with intent ("build X" / "do this properly") → `harnessed-development` sequences the gates.
2. The new **`compounding`** step (shipped today) will, at spec-close, propose updates to a living `docs/architecture/CODE-MAP.md` — which this baseline can seed.
3. `ARCHITECTURE-INVARIANTS.md` already exists — verify it names the real convergence points (auth, data access, audit) so `/code-review` can flag bypasses like the impersonation one above mechanically.
